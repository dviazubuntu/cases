The Problem This Module Solves

Clients need a fast, evidence-backed answer to "what of ours is already exposed?" — without manually querying six different intelligence tools themselves. The module lets a client submit either a domain or an uploaded file/photo, dispatches that input to five specialised external intelligence APIs in parallel, aggregates and normalises the findings, computes a 0–100 digital risk score, and shows the result as a simple dashboard the client can export. Once enrolled, the scan re-runs automatically every month with a basic new-vs-resolved delta.

Technical Architecture

A small React portal, a stateless serverless backend, and a single data store.

Layer 1: React Portal

  • /scan — landing page: scan history, new-scan button.
  • /scan/new/domain and /scan/new/file — the two input forms.
  • /report/<scan_id> — the dashboard, rendered from Firestore, exportable via the browser's print-to-PDF.
  • Firebase Authentication for login; Firestore security rules enforce tenant isolation.

Layer 2: Async Processing — Cloud Tasks

  • Scan submission queues one Cloud Task per intelligence source (5 per scan).
  • Aggregation function runs once all five report back or time out — normalises, scores, writes to Firestore.
  • 30-second per-API timeout, 3 retry attempts (Cloud Tasks' built-in retry policy); a failing source doesn't abort the scan.
  • Cloud Tasks autoscales — no fixed worker pool to size or manage.

Layer 3: External Intelligence Integration

  • Six adapters, each wrapping its own auth style — API-key header, OAuth2 token, API key + email param, or no auth (tgstate).
  • Domain path calls all five APIs directly; file path routes Vision-extracted entities through the identical five calls.
  • Per-tenant encrypted API-key vault (Secret Manager).

Layer 4: Data Layer — Firestore Only

  • Every finding, scan status, and dashboard payload lives in Firestore, scoped per tenant.
  • Delta for monthly monitoring is computed by comparing the current scan's findings to the immediately prior scan — no separate analytics warehouse.

Layer 5: Dashboard & Export

  • Six panels: Digital Risk Score gauge, Risk by Category bar, Findings by Source donut, Exposed Data Types list, Top Risk Cases cards, plain-text Summary panel.
  • Client exports the on-screen dashboard to PDF via the browser's native print function — no server-side rendering component to build or maintain.

Layer 6: Security & Retention

  • Firestore security rules enforce per-tenant scoping; every request is authenticated and scoped to a tenant_id.
  • All external API keys encrypted at rest in Secret Manager, never exposed in frontend responses or logs.
  • We recommend a focused penetration test before go-live — scoped separately from this build.
