⚙ Принцип работы

От исходящего звонка до результата

Детерминированный конвейер из 4 шагов — каждый звонок полностью прослеживается от SIP-триггера до результата в CRM.

1
SIP-триггерCRM передаёт аккаунт и номер → балансировщик нагрузки инициирует исходящий звонок через SIP Trunk
→
2
AI-переговорыLLM приветствует, идентифицирует должника, предлагает планы платежей, обрабатывает эмоциональные реакции
→
3
Определение намеренияОбещание оплаты, отказ, спор или запрос оператора-человека
→
4
ДействиеОбновление CRM/Task с журналом результата; либо передача оператору-человеку с кратким описанием контекста
⇄ Корпоративная интеграция

Бесшовная интеграция с существующей инфраструктурой

Построено на открытых стандартах — встраивается в существующие настройки PBX / оператора связи и платформы CRM без изменения архитектуры.

📞

Интеграция SIP Trunk

Протокол: SIP (RFC 3261)
Кодеки: G.711, G.722, Opus
Защищённый SIP: TLS / SRTP
DTMF: RFC 2833, SIP INFO

Инициация исходящих звонков + передача в очередь оператора-человека с сохранением контекста.

🔄

API для CRM и Task Management

REST API + MCP (Model Context Protocol). Двунаправленный обмен: получение данных аккаунта → фиксация обещания оплаты → создание задачи для дальнейшей работы.

POST /v1/call/initiate
GET /v1/account/{id}
POST /v1/disposition
POST /v1/escalate
