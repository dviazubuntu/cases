Integration

Every protocol you need to connect and receive translation

Stream audio in over one connection; get transcripts, translations, live subtitles, and synthesized speech back on the same channel.

Connect & send OUTBOUND

Open a session and stream audio to the service.

  • WebSocketWSS — the primary interface. One persistent, bidirectional connection per session for streaming.
  • RESTHTTPS — session lifecycle: create session, mint short-lived token, set language pair, read usage.
  • SDKJS / TS client SDK — wraps mic capture, the WebSocket protocol, and audio playback. Optional; the raw protocol is fully documented.
  • AuthToken-based — your backend mints a short-lived session token via REST; the client connects with it. API keys stay server-side.
  • Audio inPCM 16 kHz mono or Opus, streamed in small frames over the WebSocket.

Receive translation INBOUND

Streamed back on the same WebSocket as it's produced.

  • partialInterim transcript — live ASR text that updates as the speaker talks.
  • finalFinal transcript — settled source text for each utterance.
  • translationTranslated text in the target language, streamed incrementally.
  • subtitleLive captions — timed subtitle events (WebVTT-compatible) for both source and target.
  • audioTTS speech — synthesized target-language audio streamed back as binary frames.
  • statusJSON control — session state, language detection result, and error events.
