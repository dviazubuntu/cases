Section 01
Executive Summary

SoftTeco proposes a full delivery engagement for the NBP SharePoint Governance, Archiving & Data Optimization initiative, resourced by one person covering both Solution Architect and Developer responsibilities. The implementation approach is Power Automate-first: standard connectors, SharePoint actions, and configuration are used wherever possible, with custom code reserved for genuine gaps. This proposal replaces the earlier SA-only estimate with a re-planned, single-resource delivery model spanning design, build, migration, validation, and hypercare — scoped to the essential SA and development activities explicitly requested in the SOW.

Section 04
Detailed Phase Plan

Five delivery phases take the engagement from current-state assessment through go-live and hypercare.

1
Assessment & Strategy Definition
72h
  • Current-state assessment for storage, structure, metadata, and permissions
  • Archive eligibility criteria and governance approach definition
  • SharePoint archiving/governance strategy note and architecture decisions note
  • Scope boundaries confirmed for Archive Folder-based routing and superseded status behavior
Includes: Discovery & Requirements workshop (Workshop #1).
2
Automation Design & Development
154h
  • Workstream 1 automation: check archive condition, read Archive Folder metadata, resolve destination library, move documents without content alteration, preserve metadata/Term Store mappings, assign Superseded/Archived status
  • Workstream 3 automation: weekly trigger for K1–K4 folders, keep latest 3 versions and delete older versions only, with no change to current version/file content/permissions/metadata
  • Retries, error handling, logging, and security context configured for all flows
Largest workstream: the core Power Automate build for both metadata-driven archiving and version cleanup.
3
Archive Site Provisioning
76h
  • Dedicated Archive Site designed and configured with isolated governance settings
  • Libraries, permissions model, metadata structure, and Term Store alignment configured
  • Archive-specific retention/governance controls and superseded status behavior configured
  • Preservation requirements validated: permissions, authorship, auditability, metadata, Term Store
Delivers: the dedicated archive site and its governance/permissions foundation.
4
Data Migration & Version Cleanup
104h
  • Phased migration waves planned — pilot then controlled batches
  • Migration of historical/inactive data executed with folder hierarchy and metadata preservation
  • Version cleanup automation for K1–K4 run as part of phased execution
  • Post-migration reconciliation and rollback readiness checks performed
Risk-managed execution: pilot batch validated before full-scale migration proceeds.
5
Testing, Validation & Go-Live
74h
  • End-to-end functional tests executed with UAT support
  • Data integrity and automated superseded/archive status assignment validated
  • Post-migration preservation of metadata, permissions, authorship, and audit trail verified
  • Testing/validation/audit report finalized; go-live readiness gate run; transition/hypercare supported
Includes: Go-Live Readiness Gate workshop (Workshop #2).

Section 07
Assumptions

This estimate holds under the following assumptions. Material deviations may require a re-baseline of scope, schedule, or cost.

  • Required Dev/Test/Prod access and permissions are granted on time.
  • Term Store and required metadata fields exist, or are remediated early.
  • Power Automate licensing and connectors are available.
  • SMEs and key stakeholders attend critical workshops and provide timely approvals.
  • Migration windows and operational support are available for pilot and cutover.
